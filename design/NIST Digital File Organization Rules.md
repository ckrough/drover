**Character and Path Restrictions**
- File paths must not exceed 255 characters total
- File names should be approximately 25 characters
- Folder hierarchy must not exceed eight levels
- Use only characters a-z, numbers 0-9, underscores (_), and hyphens (-)
- Do not use spaces in file names
- Do not use special characters: . \ / : * ? " < > | [ ] & $ ,
- File names must contain only one period, immediately before the file extension
- Folder names must not contain periods

**Naming Structure**
- Order file name elements from general to specific
- Maintain consistent element positioning across all files of the same type (e.g., report_name_yyyymmdd)
- Use leading zeros for numerical sequences (001, 002, 010, 011, 100)
- Use lowercase letters with uppercase at the start of each word (File_Name_Convention_001.doc)
- Use Title Case for folder names
- Avoid filler words: a, and, of, the, to
- Use standard abbreviations per GPO Style Manual

**Dates**
- Embed relevant dates directly in file names
- Use ISO 8601 format: YYYYMMDD
- Place date at end of file name

**Descriptive Requirements**
- File names must be understandable independent of folder location
- File names must be meaningful to users other than the creator
- Folder names must enable retrieval without prior knowledge of contents
- Do not use ambiguous terms: update, new, old

**Organizational Structure**
- Model folder structure on organizational functions or subject themes
- Minimize records per folder to maintain usability
- Apply naming conventions uniformly
- Document and enforce organizational naming decisions
